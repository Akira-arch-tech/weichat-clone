
# Demo Script（3-5 分钟演示脚本）

## 0. 演示前准备（1分钟）

- [ ] 后端已启动：`http://localhost:3001`
- [ ] 前端已启动：`http://localhost:5173`
- [ ] 演示账号可登录
- [ ] 浏览器窗口尺寸接近移动端（建议宽度 390px）

---

## 1. 开场（20秒）

话术示例：
> 这是一个 React + Node.js 的微信核心流程 demo，目标是展示前后端协作和关键交互闭环，包括登录、聊天、通讯录和朋友圈。

---

## 2. 流程一：登录与会话（60秒）

步骤：
1. 使用演示账号登录
2. 展示会话列表（头像、昵称、最后消息）
3. 进入某个会话并发送一条文本消息

话术重点：
- 前端负责交互和展示
- 后端负责鉴权和消息存储
- Socket 实现实时消息刷新

---

## 3. 流程二：通讯录与好友关系（45秒）

步骤：
1. 切到通讯录
2. 搜索联系人
3. 展示好友申请流程（发送/接收）

话术重点：
- 这是“用户关系链”的基础数据
- 关系建立后才进入聊天场景

---

## 4. 流程三：朋友圈（60秒）

步骤：
1. 进入发现 -> 朋友圈
2. 发布一条文字动态（可附图）
3. 展示点赞或评论

话术重点：
- 朋友圈是“内容分发与互动”场景
- 同样依赖前后端接口和数据一致性

---

## 5. 流程四：我与设置（40秒）

步骤：
1. 进入“我”
2. 打开设置，编辑昵称或签名并保存

话术重点：
- 说明账号与资料配置能力
- 体现完整产品闭环（使用->管理）

---

## 6. 收尾（20秒）

话术示例：
> 这个 demo 的重点不是 100% 还原微信，而是用最小功能集证明：产品流程可跑通、前后端可协作、核心体验可验证。

---

## 7. 备用方案（防翻车）

- 如果实时消息异常：改为演示会话列表与历史消息
- 如果图片上传异常：改演示文本消息和朋友圈文字发布
- 如果网络慢：先演示已加载页面，再补讲技术架构
