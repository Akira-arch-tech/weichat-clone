# PRD（一页纸）- WeChat Clone Demo

## 1. 项目目标（Why）

做一个可运行、可交互、可演示的微信核心流程 demo，用于：
- 全栈协作关系学习（前端-后端-数据库-实时通信）
- 产品经理面试展示（需求拆解 + 流程设计 + demo 落地）

---

## 2. 目标用户（Who）

- 主要用户：学习产品/技术协作的个人（PM 视角）
- 次要用户：面试官、评审、协作工程师

---

## 3. 核心场景（What）

1. 用户 A 登录后进入会话列表
2. 用户 A 与用户 B 实时聊天（文本）
3. 用户 A 查看通讯录并发起好友申请
4. 用户 A 在朋友圈发布动态，用户 B 点赞/评论
5. 用户 A 在“我-设置”编辑个人资料

---

## 4. MVP 范围（本期必须）

### Must Have
- 登录 / 注册
- 会话列表 + 1v1 聊天（文本）
- 通讯录 + 好友申请
- 朋友圈发布与互动（点赞/评论）
- 个人资料查看与编辑

### Nice to Have（可选）
- 图片消息
- 朋友圈多图发布
- 更细致的失败态和空态

### Out of Scope（不做）
- 支付、红包、转账
- 群聊复杂权限
- 音视频通话
- 生产级安全与高并发优化

---

## 5. 成功标准（Success Metrics）

- 能在 3 分钟内完整演示 1 条主流程（登录→聊天→朋友圈）
- 关键页面可访问率 100%（无死链）
- 主要操作响应正常（登录、发送消息、发布动态）

---

## 6. 信息架构（IA）

- Tab1：微信（会话列表、聊天页）
- Tab2：通讯录（联系人、申请、详情）
- Tab3：发现（朋友圈入口等）
- Tab4：我（个人信息、设置）

---

## 7. 数据对象（PM 最小粒度）

- 用户 User
- 会话 Conversation
- 消息 Message
- 好友关系 Friendship
- 好友申请 FriendRequest
- 朋友圈动态 Moment
- 点赞 Like
- 评论 Comment

---

## 8. 风险与约束

- clone 项目仅用于学习演示，避免商标误导
- 演示数据与真实生产数据隔离
- 账号与密钥信息不写入公开仓库

---

## 9. 里程碑（建议）

- M1：环境跑通 + 基础导航（D1-D2）
- M2：聊天主流程可演示（D3-D4）
- M3：朋友圈与个人设置补齐（D5-D6）
- M4：文档完善 + 演示彩排（D7）

---

## 10. 验收清单（Checklist）

- [ ] 本地可启动（前后端）
- [ ] 演示账号可登录
- [ ] 文本消息可发送/接收
- [ ] 朋友圈可发布并看到互动
- [ ] README 和 demo 脚本完整
